title: 任务完成情况定期评审
description: This guide provides instructions on customizing our application.
---
技术委员会定期评审开源项目发布的任务完成情况、导师辅导学生情况等，根据评审结果放宽或收缩其任务发布限制。


## 评审标准
技术委员会制定项目任务完成情况评审标准，包括：
1. 任务是否有导师审核；
2. 代码质量：
3. 导师是否有辅导学生；
4. 学生反馈好坏与否等。
## 初期评审
项目发布任务1-3个月后，技术委员会相关工作组对项目进行初期评审。
初期评审通过的项目，放宽任务发布限制，包括分值与数量限制；单个任务最大分值为20分；同时进行的任务数量为3~6个。
## 中期评审
项目发布任务3-6个月后，技术委员会相关工作组对任务完成情况进行评审。
中期评审通过的项目，放宽任务发布限制，包括分值与数量限制；单个任务最大分值30分；同时进行的任务数量为6~10个。
## 持续评审
技术委员会每隔3个月对项目任务完成情况再次评审，并逐步放宽任务分值与数量限制。
