任务是通过 GitHub Issue 来管理的，所有的命令都在 Issue 的评论中完成。

### 实习学生使用的命令

| 命令名称          | 命令说明                                                                         |
| ----------------- | -------------------------------------------------------------------------------- |
| /request-assign   | 实习学生申请 **开放** 状态的实习任务，任务状态变为 **认领审核**                  |
| /request-complete | 实习任务编码完成后，申请实习任务完成；任务将从 **已认领** 变为 **完成审核** 状态 |
| /request-release  | 学生放弃申请任务，或放弃一个已经分配的实习任务，任务将变为 **开放**              |

### 导师命令使用的命令

| 命令名称           | 命令说明                                                                                         |
| ------------------ | ------------------------------------------------------------------------------------------------ |
| /intern-approve    | 导师同意实习学生认领任务，任务将从 **认领审核** 变为 **已认领**                                  |
| /intern-disapprove | 导师拒绝认领，任务将从认领审核变为 **开放**                                                      |
| /intern-fail       | 导师认为实习学生无法在按时完成任务，终止该学生的任务，任务将变为 **开放**                        |
| /intern-done       | 导师合入实习学生提交的 PR ，确认任务完成                                                         |
| /intern-close      | 导师认为当前 Issue 已经不合适作为实习任务，如任务已经由其它开发者完成等 |
