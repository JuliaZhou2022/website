### 发布任务的限制

<Note type="danger" title="任务数量和分值限制">
    1. 开源项目加入 R2CN 计划后，单个任务的分值限制为 10 分；任务分数可以参照难度、所需完成时间来确定，具体可参考 [任务分值规则](/docs/project/mentor-and-tasks/score)；
    2. 开源项目加入 R2CN 计划后，同时发布的任务数量限制为 2 个；如果开源项目参加过开源之夏，同时发布的任务数量限制为 3 个；
    3. 开源项目发布的任务仅可允许一位学生领取，学生在完成任务后才能领取新任务；
    4. R2CN 技术委员会会对开源项目进行定期 Review ，根据开源项目任务的完成情况和辅导情况，调整单个任务的分值和同时发布任务的数量；
</Note>


### 发布任务的建议

<Note type="note" title="发布任务的建议">
    1. 尽量将工作拆分为比较小的任务单元，以便学生能够更好地理解和完成工作；当学生多次顺利完成任务后，可以将拆分粒度变为更大的任务单元；
    2. 发布任务时尽量使用模板任务的[模板](/docs/project/mentor-and-tasks/template)，也可以根据开源项目的实际情况调整模板；
    3. 对任务的描述一定要尽量详细；如果是研究性的任务，一定要对研究方向给予明确的说明；
</Note>


### 平台限制

由于目前 [R2CN-Bot](/docs/project/bot) 还没有适配 GitHub 以外的代码托管平台，任务在 GitHub 上的开源项目代码仓库发布。
