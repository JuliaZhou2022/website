title: 任务Issue模板

---

### 创建模版

本文档仅列出了部分步骤，更加详细的文档可参考 [在Github 新建Issue 模版](https://docs.github.com/en/communities/using-templates-to-encourage-useful-issues-and-pull-requests/configuring-issue-templates-for-your-repository)

<Stepper>
  <StepperItem title="第一步：进入项目设置">
    访问项目的主页，确认有Admin权限，点击Settings
    <img src="/images/issue_template-1.jpg" />
  </StepperItem>
  <StepperItem title="第二步：设置模版">
    找到Issue部分，点击Set up templates
    <img src="/images/issue_template-2.jpg" />
  </StepperItem>
  <StepperItem title="第三步：自定义模版内容">
    选择自定义模版内容，输入以下模版示例中的例子，根据需要添加自定义标题和标签
    <img src="/images/issue_template-3.jpg" />
  </StepperItem>
  <StepperItem title="第四步：提交自定义模版">
    <img src="/images/issue_template-4.jpg" />
  </StepperItem>
  <StepperItem title="第五步：新建Issue">
    在Issue tab下新建Issue，查看刚才添加的模版是否正确
    <img src="/images/issue_template-5.jpg" />
  </StepperItem>
</Stepper>

### 模版示例

  <Tabs defaultValue="cn" className="pt-5 pb-1">
    <TabsList>
      <TabsTrigger value="cn">中文模版</TabsTrigger>
      <TabsTrigger value="en">En</TabsTrigger>
    </TabsList>
    <TabsContent value="cn">
```txt
[标题] 为xxx实现xxx功能
[任务分值] xx分（根据任务分级自评）
[背景描述] 描述产品及特性
[需求描述] 目前需要开发的特性
[环境要求] 描述开发环境
[产出标准] xxx
[PR 提交地址] 仓库xxx分支的xxx目录
[期望完成时间] 202x年xx月xx日
[开发指导] 参考资料等
[导师及邮箱] xxx
[备注] xxx
[标签] xxx（在对应的代码托管平台上使用 r2cn 的标签）
```
    </TabsContent>
    <TabsContent value="en">
    ```txt
[Title] Implementing xxx functions for xxx
[Task Score] xx points (self-assessment based on task grading)
[Background Description] Describe the product and its features.
[Description of Requirements] Describe the features that need to be developed at this time.
[Environmental Requirements] Describe the development environment.
[Output Criteria] xxx
[PR Submission Address] xxx directory in xxx branch of the repository
[Desired Completion Date] xx/xx/202x
[Development Guidance] References, etc.
[Mentor and e-mail] xxx
[Remarks] xxx
[Tags] xxx (use r2cn's tags on the corresponding code hosting platform)
```
    </TabsContent>
</Tabs>
