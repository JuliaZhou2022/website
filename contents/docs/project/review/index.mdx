### 技术委员会陈述

<Stepper>
  <StepperItem title="技术委员会会议陈述">
      技术委员会向开源项目申请加入 Issue 中的联系方式发送会议通知，通知开源项目参加技术委员会会议，对项目进行陈述。
      <Note type="danger" title="陈述注意事项">
          1. 请使用类似腾讯文档等服务编写陈述使用的材料，说明项目情况即可，不限制格式和形式，材料访问设置为公开可访问；
          2. 陈述材料准备好后请在申请加入的 Issue 中添加一个 Comment， `@` 技术委员会秘书 `genedna`，添加材料的在线地址；
          3. 材料中至少要阐述项目属于云原生技术栈理由、RISC-V 适配计划；
      </Note>
  </StepperItem>
  <StepperItem title="技术委员会成员投票">
      陈述后，技术委员会成员对项目进行投票，决定是否通过项目加入到 R2CN 计划。
      <Note type="danger" title="投票规则">
          1. 技术委员会成员 1/2 及以上成员投赞成票即可通过；
          2. 技术委员会成员在开源项目申请加入 Issue 回复 +1 即为投赞成票；
      </Note>
  </StepperItem>
</Stepper>

### 安装 R2CN-Bot

开源项目通过技术委员会的投票通过后，开源项目仓库需要安装 [R2CN-Bot](/docs/project/bot)，请参考文档中 [安装 Bot](/docs/project/bot/installation) 部分进行安装。

### 添加仓库和导师

通过评审的开源项目，向 R2CN 仓库提交代码仓库和导师，添加成功后导师可以开始发布任务。

<Stepper>
  <StepperItem title="第一步：Fork 仓库">
    请 Fork [r2cn-dev/r2cn](https://github.com/r2cn-dev/r2cn) 到开源项目的 GitHub 组织下；
  </StepperItem>
  <StepperItem title="第二步：修改 r2cn.yaml, 添加仓库地址和导师信息">
    ```yaml
    - name: "仓库的组织/名称"
      maintainers:
      - id: "导师 GitHub ID"
        task: 2
        maxScore: 10
    ```

    提交信息示例：

    ```yaml
    repos:
    - name: "web3infra-foundation/mega"
      maintainers:
      - id: "benjamin-747"
        task: 3
        maxScore: 20
      - id: "genedna"
        task: 3
        maxScore: 20
    ```

    <Note type="danger" title="注意事项">
      1. 如果一个开源项目有多个代码仓库，请在 r2cn.yaml 中添加多个仓库；
      2. 一个仓库可以有多个导师，`maintainers` 要使用导师的 GitHub ID，不是邮箱地址；
      3. Maintainer 必须拥有仓库的写权限，有权限批准 PR 的合入；
      4. 开源项目初期同时发布任务数量限制为 **2** ，参加过开源之夏的开源项目同时发布任务数量初始为 **3** ；R2CN 技术委员会会定期评审任务完成和辅导质量，根据情况会逐步放宽同时发布任务的数量限制；
    </Note>
  </StepperItem>
  <StepperItem title="提交 PR 到 r2cn-dev/r2cn ">
    修改完成后提交 PR 到 [r2cn-dev/r2cn](https://github.com/r2cn-dev/r2cn) 仓库，等待技术委员会审核通过后合入 PR ，导师才可以发布实习任务。
  </StepperItem>
</Stepper>
